Mise en place de procédure pour l'application de ces tests

La procedure d'automatisation des tests.

Dans le cadre des processus d'integration continue, une automation de tests peut être mise en place.

Des outils comme Jenkins peut-être appliqué pour mettre en place les differents tests dans un pipeline (XXXXX)

Les codes ne doivent pas être fusionner sur branche "main" tant que les tests ne passent pas.

Les developpeurs sont solicités * à passer leur codes à travers de ces differents tests * et à ne pas merger dans la branche commun de developpement tant que les tests ne sont pas passés positivement.
